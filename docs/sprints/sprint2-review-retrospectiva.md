## Retrospectiva

### O que correu bem
- Comunicação da equipa activa

### O que correu mal
- ORM não identificado como requisito obrigatório desde o início
- Sprint a incluir US com dependências
- Branch criada a partir de outra branch em vez da main, como resultado de US dependentes distribuídas entre diferentes membros da equipa
- Migração para Prisma identificada tarde
- Frontend não iniciado — sem interface visual para demonstrar

### O que vamos melhorar no Sprint 3
- Migração Prisma como primeira prioridade
- Cada developer cria branch sempre a partir da main
- Identificar dependências técnicas antes de começar a programar
- Configurar Design System (shadcn/ui + Tailwind) antes de iniciar qualquer ecrã frontend
- Primeiro ecrã frontend funcional até ao final do Sprint 3
